Made the stub `TailscaleBackend` in `AppModule.kt` stateful so it tracks its running state. After `start()` is called, `isRunning()` returns `true` and `getIp()` returns a placeholder IP (`100.100.100.100`). This prevents `GrpcServerService.startServer()` from entering the stale-auth path that repeatedly calls `handleStaleAuth()` -> `clearAuthKey()`, which was destroying the user's stored auth key in a loop. The gRPC server still can't receive real sign requests from the host (no real Tailscale), but the auth destruction loop is broken and the service lifecycle is stable. For actual E2E sign request testing, BUG-015's test-sign deep link bypass is the solution.
