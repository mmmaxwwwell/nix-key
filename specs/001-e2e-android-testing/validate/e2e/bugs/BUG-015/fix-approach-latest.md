Added a debug-only `nix-key://test-sign` deep link handler that injects fake sign requests into `SignRequestQueue` for E2E UI testing without requiring a real Tailscale connection. The intent filter is registered in the debug-only `AndroidManifest.xml` (alongside the existing `nix-key://pair` handler). `MainActivity` handles the deep link in both `onCreate` and `onNewIntent`, gated behind `BuildConfig.DEBUG`, by constructing a `SignRequest` from query parameters (host, key, fingerprint, unlock) and calling `signRequestQueue.enqueue()`. Added `injectSignRequest()` to `NixKeyE2EHelper.kt` for programmatic use from instrumented tests. This follows the exact same pattern as the existing pairing deep link bypass and enables full E2E validation of the SignRequestDialog approve/deny flows.
