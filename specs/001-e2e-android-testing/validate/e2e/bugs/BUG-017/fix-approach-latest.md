Added `ensureServerCertExists(alias)` method to `KeyManager.kt` that checks if the Android Keystore already contains an entry at the given alias and, if not, generates a new ECDSA-P256 key pair using `KeyPairGenerator` with `KeyGenParameterSpec`. Android Keystore automatically creates a self-signed X.509 certificate when generating a key pair, so this populates the certificate that `getServerCertPem()` later exports. The cert is configured with subject CN=nix-key phone server and a 10-year validity. In `PairingViewModel.startPairing()`, `keyManager.ensureServerCertExists(serverCertAlias)` is called immediately after `getOrCreateServerCertAlias()` and before `getServerCertPem()`, ensuring the certificate exists before attempting to export it as PEM. This is different from the previous fix which only added `getServerCertPem()` (export) but never addressed the missing certificate generation.
