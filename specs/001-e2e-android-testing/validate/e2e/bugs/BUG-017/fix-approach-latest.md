Added `getServerCertPem(alias)` method to `KeyManager.kt` that loads the certificate from Android KeyStore via `keyStore.getCertificate(alias)`, encodes its DER bytes to Base64, and wraps with PEM headers. Injected `KeyManager` into `PairingViewModel` (already a Hilt `@Singleton`) and in `startPairing()`, after obtaining the server cert alias, call `keyManager.getServerCertPem(serverCertAlias)` to get the actual PEM string. Pass this PEM string (instead of the alias string) to `pairingClient.pair()` as `phoneServerCert`. The alias is still stored in `PairedHost.phoneServerCertAlias` for local KeyStore lookups. This ensures the host receives a valid PEM certificate it can decode and fingerprint.
