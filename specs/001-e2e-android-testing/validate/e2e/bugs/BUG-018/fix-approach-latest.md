Added a `LifecycleResumeEffect(Unit)` to `ServerListScreen.kt` that calls `viewModel.refresh()` every time the screen's lifecycle enters the RESUMED state. This covers the case where `popBackStack()` returns to an existing ServerListScreen after deep link pairing — the ViewModel is reused (its `init` block doesn't re-run) but the LifecycleResumeEffect fires and reloads the host list from `HostRepository`. The `lifecycle-runtime-compose:2.8.7` dependency (already declared) provides this API. No changes to ViewModel, HostRepository, or NavGraph were needed.
