Added `LifecycleResumeEffect` to `KeyListScreen.kt` so that `viewModel.refresh()` is called every time the screen's lifecycle transitions to RESUMED — including when navigating back from KeyDetail after creating a key. This mirrors the existing pattern in `ServerListScreen.kt` (which uses the same `LifecycleResumeEffect` to refresh the host list on resume). Previously, `refresh()` was only called in `KeyListViewModel.init{}`, so the key list stayed stale when the ViewModel was retained on the back stack during forward navigation.
