Replaced the hard `throw IllegalStateException("Tailscale not connected")` in `PairingViewModel.kt:120` with a fallback to `payload.host` when `tailscaleManager.getIp()` returns null. The Tailscale IP is only used in the POST body to tell the host where to reach the phone later — it is not used for the pairing HTTP connection itself. Using `payload.host` as a fallback is safe because in the emulator/E2E context the host can reach the phone at that address. A warning is logged via Timber when the fallback is used.
