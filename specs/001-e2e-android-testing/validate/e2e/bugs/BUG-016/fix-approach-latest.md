Added stop-before-start guards in `TailscaleAuthViewModel.connectWithAuthKey()` and `connectWithOAuth()`. Before calling `tailscaleManager.start()`, both methods now check `tailscaleManager.isRunning()` and call `tailscaleManager.stop()` if true. This handles the race condition where `GrpcServerService` auto-starts the TailscaleManager (without an auth key) before the user taps Connect, which previously caused an `IllegalStateException("TailscaleManager is already running")`. The fix mirrors the existing pattern in the `retry()` method in the same ViewModel.
