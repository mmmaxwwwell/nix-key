Two changes: (1) In `MainActivity.handleTestSignDeepLink()`, replaced the hardcoded `ConfirmationPolicy.ALWAYS_ASK` with a `policy` query parameter parsed from the deep link URI. E2E tests can now use `policy=AUTO_APPROVE` to skip BiometricPrompt entirely on emulators without screen lock. Invalid values fall back to `ALWAYS_ASK`. (2) In `BiometricHelper.showPrompt()`, added a `canAuthenticate()` pre-check before calling `prompt.authenticate()`. If the device doesn't support the requested authenticator (e.g., no PIN set for DEVICE_CREDENTIAL), it returns `AuthResult.Failure` immediately with a clear log message instead of letting BiometricPrompt silently fail with error code 14. (3) Updated `NixKeyE2EHelper.injectSignRequest()` to accept a `policy` parameter defaulting to `AUTO_APPROVE` for E2E test compatibility.
