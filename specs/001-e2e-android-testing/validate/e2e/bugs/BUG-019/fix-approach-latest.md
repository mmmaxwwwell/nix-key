Changed the default confirmation policy in `MainActivity.handleTestSignDeepLink()` from `ALWAYS_ASK` to `AUTO_APPROVE`. Since this method is debug-only (`BuildConfig.DEBUG` guard) and exists solely for E2E testing, defaulting to `AUTO_APPROVE` avoids requiring device credentials on emulators that don't have a screen lock configured. The `policy` query parameter still works for explicitly requesting `ALWAYS_ASK` or other policies when needed. Additionally, in `proceedWithSigningConfirmation()`, added a Toast message when authentication fails (AuthResult.Failure), so the user gets visible feedback explaining that a screen lock must be set — this addresses the silent denial UX issue where the dialog would close without any explanation. This is different from the previous fix which added the `policy` parameter support but kept the default as `ALWAYS_ASK`, causing the test-sign deep link to still require device credentials by default.
