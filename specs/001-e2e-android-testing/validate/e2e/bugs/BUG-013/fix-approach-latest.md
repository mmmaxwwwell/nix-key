Fixed the post-pairing navigation failure with two changes. First, in `NavGraph.kt`, made `onPairingComplete` resilient by adding a fallback: if `popBackStack(SERVER_LIST)` fails (returns false because SERVER_LIST isn't in the back stack during deep link flows), it navigates to SERVER_LIST while clearing the entire back stack via `popUpTo(graph.id, inclusive=true)`. This handles both the normal flow (popBackStack succeeds) and the deep link flow (fallback navigate). Second, added `android:launchMode="singleTop"` to MainActivity in `AndroidManifest.xml` so deep link intents are delivered via `onNewIntent()` instead of destroying and recreating the activity, preserving the existing navigation state including SERVER_LIST in the back stack.
